**NavigationContext.tsx** — Create `src/contexts/NavigationContext.tsx`:

- Type `NavDirection = 'tab' | 'forward' | 'back'`
- Context holds `direction: React.MutableRefObject<NavDirection>` and `navigateTo: (path: string, direction?: NavDirection) => void`
- `NavigationProvider` wraps children, uses `useRef<NavDirection>('tab')` and `useNavigate()` internally
- `navigateTo` sets `direction.current = dir` then calls `navigate(path)`
- `useNavigation()` hook reads context, throws if outside provider
- Use a ref (NOT state) for direction — state would cause unnecessary re-renders on every navigation, and variants are read synchronously at render time

Follow the exact pattern from 02-RESEARCH.md Pattern 2.

**AppShell.tsx** — Make direction-aware:

1. Import `useNavigation` from `../../contexts/NavigationContext`
2. Import `slideRight`, `slideLeft` from `../../motion/variants`
3. Read `direction` ref from `useNavigation()`
4. Compute `pageVariants` at render time (not module level): `direction.current === 'tab' ? fadeIn : direction.current === 'forward' ? slideRight : slideLeft`
5. Pass `pageVariants` to the `m.div` variants prop (replacing the hardcoded `fadeIn`)
