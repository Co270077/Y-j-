Create `src/hooks/useSwipeAction.ts` implementing the swipe gesture logic:

Interface:
```typescript
interface UseSwipeActionOptions {
  onComplete?: () => void   // swipe-right callback (e.g., complete task)
  onDelete?: () => void     // swipe-left callback (e.g., delete item)
  threshold?: number        // 0-1, default 0.4 (40% of item width)
  enabled?: boolean         // default true, allows disabling gesture
}

interface UseSwipeActionReturn {
  bind: (...args: any[]) => any  // spread onto gesture container
  x: MotionValue<number>         // drives item translateX
  rightBgOpacity: MotionValue<number>  // green background opacity (0-1)
  leftBgOpacity: MotionValue<number>   // red background opacity (0-1)
  isSwiping: boolean              // true during active gesture (for disabling tap)
  containerRef: RefObject<HTMLDivElement | null>  // ref to measure width
}
```

Implementation:
- `useMotionValue(0)` for x offset
- `useRef` for `directionLocked` (left | right | null) and `hasTriggeredHaptic` (boolean)
- `useRef` for container element to measure width via `containerRef.current?.offsetWidth`
- Compute threshold dynamically: `(containerRef.current?.offsetWidth || 375) * thresholdPct`
- `useTransform(x, [0, THRESHOLD], [0, 1])` for rightBgOpacity
- `useTransform(x, [-THRESHOLD, 0], [1, 0])` for leftBgOpacity
- `useDrag` with `{ axis: 'lock', filterTaps: true, pointer: { capture: false } }`:
  - On first 10px: check `|my| > |mx|` — if true, call `cancel()` (vertical scroll priority)
  - Lock direction after 10px horizontal: `directionLocked.current = dx > 0 ? 'right' : 'left'`
  - Prevent opposite direction: if locked right, clamp mx >= 0; if locked left, clamp mx <= 0
  - Only allow swipe-right if `onComplete` is provided; only allow swipe-left if `onDelete` is provided
  - Apply rubber-band resistance past threshold: `THRESHOLD + (mx - THRESHOLD) * 0.3`
  - Haptic at threshold crossing: call `hapticSuccess()` once when `|mx|` first exceeds threshold (track via ref)
  - On release (`!active`):
    - If past threshold AND correct direction: call `onComplete`/`onDelete`, then animate x to `+/- window.innerWidth` (fly off), return
    - If below threshold: `animate(x, 0, { type: 'spring', stiffness: 400, damping: 28 })`
  - Reset `directionLocked` and `hasTriggeredHaptic` on gesture end
- Track `isSwiping` via `useState` — set true on drag start, false on end
- Return `{ bind, x, rightBgOpacity, leftBgOpacity, isSwiping, containerRef }`

Critical:
- Import `useMotionValue`, `useTransform`, `animate` from `'motion/react'` (NOT `motion/react-m`)
- Do NOT call `animate()` during active drag — only `x.set()` during drag, `animate()` on release
- `pointer: { capture: false }` allows browser scroll to take over

Implementation:
- Call `useSwipeAction({ onComplete, onDelete, enabled })`
- Render structure (3 layers):
  1. **Outer container** (`div`): `ref={containerRef}`, `{...bind()}`, `position: relative`, `overflow: hidden`, `rounded-[var(--radius-md)]`. This receives gesture events.
  2. **Background reveal layer** (absolute positioned behind content):

     - Both backgrounds render always but are invisible (opacity 0) at rest
  3. **Content layer**: `m.div` with `style={{ x }}` — this is the child content that visually slides. Background `bg-surface-raised` to cover the reveal layer at rest.

- Add `touch-action: pan-y` style to outer container — lets browser handle vertical scrolling
- When `enabled` is false, do not spread `bind()` and do not render backgrounds
